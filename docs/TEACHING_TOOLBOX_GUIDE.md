## 🎯 功能概述

### 系统定位
**教学工具箱** 是一个AI赋能的教学辅助系统，专注于解决教师日常备课中的三大痛点：
- 课件案例更新
- 实验指导书编写
- 主观题作业批阅

### 核心特色
与其他AI教学系统的差异化：

| 功能模块 | 其他系统 | 教学工具箱 | 差异化优势 |
|---------|---------|-----------|-----------|
| 课件制作 | 生成新PPT | 更新现有PPT案例 | 保留教师风格，只更新内容 |
| 题目生成 | 生成选择题/填空题 | 生成实验指导书 | 侧重实践操作，而非理论测试 |
| 作业批阅 | 批阅客观题 | 批阅主观题 | AI分析论述题，提供详细评语 |

### 三大核心功能

#### 1. 📝 课件更新助手
**功能**：分析现有PPT，提供最新案例更新建议

**使用场景**：
- 每学期更新课件中的案例
- 补充最新的行业动态
- 更新过时的数据和图表

**工作流程**：
```
选择课程 → 上传PPT文件 → 描述更新要求 → AI搜索最新案例 → 生成更新建议
```

#### 2. 🔬 实验指导书生成
**功能**：根据章节和知识点生成完整的实验指导书

**生成内容**：
- 实验目的（3-5条）
- 实验环境和工具
- 详细实验步骤（含代码）
- 预期结果
- 思考题（3道）
- 参考答案

**工作流程**：
```
选择课程 → 输入章节 → 添加知识点 → 选择难度 → AI生成指导书
```

#### 3. ✍️ 主观题批阅助手
**功能**：AI辅助批阅论述题、分析题等主观作业

**批阅内容**：
- 自动评分（0-100分）
- 详细评语（100-200字）
- 优点分析
- 不足指出
- 改进建议

**工作流程**：
```
选择课程和学生 → 提交作业 → AI批阅 → 教师审阅 → 确认成绩
```

## 📖 功能详解

### 功能1：课件更新助手

#### 操作步骤
1. 点击"更新课件案例"卡片
2. **选择课程**（自动加载教师的课程）
3. **上传PPT文件**或输入文件名
4. **输入更新要求**，例如：
   ```
   1. 更新2024-2025年最新的网络安全事件案例
   2. 补充AI安全相关的前沿知识
   3. 更新数据统计图表
   ```
5. 点击"提交"，等待AI分析

#### 查看结果
- 在"课件更新记录"标签页查看
- 状态为"建议已生成"时，点击"查看建议"
- AI会提供：
  - 需要更新的页码
  - 具体章节
  - 更新建议
  - 推荐的新案例内容

#### 示例场景
```
课程：信息安全
文件：信息安全课件.pptx
要求：更新2024-2025年最新的网络安全事件案例

AI建议：
- 第5页：补充2024年某公司数据泄露事件
- 第12页：更新勒索软件攻击统计数据
- 第18页：添加AI安全威胁案例
```

### 功能2：实验指导书生成

#### 操作步骤
1. 点击"实验指导书"卡片
2. **选择课程**（自动加载教师的课程）
3. **输入章节名称**，例如：第三章：栈和队列
4. **添加知识点**（按回车添加）：
   - 栈的定义
   - 栈的应用
   - 队列的定义
   - 循环队列
5. **选择难度**：简单/中等/困难
6. 点击"生成"，等待AI生成

#### 查看结果
- 在"实验指导书"标签页查看
- 状态为"已完成"时，点击"查看"
- 完整的实验指导书包含：
  - 📌 实验目的
  - 💻 实验环境
  - 📝 实验步骤
  - 💡 示例代码
  - ✅ 预期结果
  - 🤔 思考题
  - 📖 参考答案

#### 示例场景
```
课程：数据结构
章节：第三章：栈和队列
知识点：栈的定义、栈的应用、队列的定义、循环队列
难度：中等

生成内容：
1. 实验目的：掌握栈和队列的基本概念...
2. 实验环境：JDK 1.8+, IDE: IntelliJ IDEA
3. 实验步骤：
   步骤1：创建栈的数据结构
   步骤2：实现push和pop操作
   ...
```

### 功能3：主观题批阅助手

#### 操作步骤
1. 点击"智能批阅"卡片
2. **选择课程**（自动加载教师的课程）
3. **选择学生**（可搜索）
4. **输入作业标题**，例如：数据结构第三章学习总结
5. **输入作业内容**（论述题、分析题等）
6. 可选：上传文件（Word/PDF/TXT）
7. 点击"提交"

#### AI批阅
- 在"作业批阅"标签页找到提交的作业
- 点击"AI批阅"按钮
- 等待10-30秒

#### 查看结果
- 点击"查看结果"
- AI批阅包含：
  - 📊 评分（圆形进度条显示）
  - 💬 详细评语
  - ✅ 优点分析
  - ⚠️ 不足指出
  - 💡 改进建议

#### 教师审阅
- 可以修改AI给出的评分
- 可以补充教师评语
- 点击"提交审阅"确认最终成绩

#### 示例场景
```
作业：请论述栈和队列的区别与联系

AI批阅结果：
- 评分：85分
- 评语：该作业对栈和队列的基本概念理解准确，能够从数据结构特点进行对比分析...
- 优点：
  1. 概念理解准确
  2. 举例恰当
- 不足：
  1. 缺少实际应用场景的深入分析
- 建议：
  1. 补充更多实际开发中的应用案例
```

## 🔧 技术实现

### 后端架构

#### 技术栈
- **框架**：Spring Boot 2.x
- **ORM**：MyBatis（注解方式）
- **AI**：阿里云通义千问（DashScope API）
- **数据库**：MySQL 8.0

#### 核心组件
```
Controller (9个API)
    ↓
Service (业务逻辑)
    ↓
Mapper (数据访问)
    ↓
Database (3个表)
```

#### API列表

**课件更新助手**
- `POST /api/smart-teaching/update-courseware` - 提交更新任务
- `GET /api/smart-teaching/courseware-suggestions/{id}` - 获取建议
- `GET /api/smart-teaching/courseware-updates/{teacherId}` - 获取记录

**实验指导书生成**
- `POST /api/smart-teaching/generate-experiment-guide` - 生成指导书
- `GET /api/smart-teaching/experiment-guide/{id}` - 获取内容
- `GET /api/smart-teaching/experiment-guides/{teacherId}` - 获取列表

**主观题批阅助手**
- `POST /api/smart-teaching/submit-homework` - 提交作业
- `POST /api/smart-teaching/grade-homework/{id}` - AI批阅
- `GET /api/smart-teaching/homework-list/{teacherId}` - 获取列表
- `POST /api/smart-teaching/review-homework` - 教师审阅

### 前端架构

#### 技术栈
- **框架**：Vue 3 (Composition API)
- **UI库**：Element Plus
- **HTTP**：Axios
- **路由**：Vue Router

#### 组件结构
```
SmartTeachingAssistant.vue (主页面)
├── CoursewareUpdaterDialog.vue (课件更新)
├── ExperimentGeneratorDialog.vue (实验指导书)
├── HomeworkGradingDialog.vue (作业提交)
├── SuggestionsViewDialog.vue (查看建议)
├── ExperimentViewDialog.vue (查看指导书)
└── GradingResultDialog.vue (查看批阅结果)
```

#### 核心特性
- ✅ 自动加载教师课程（`getCoursesByTeacherId`）
- ✅ 学生选择（可搜索）
- ✅ 文件上传（PPT、Word、PDF）
- ✅ 实时状态更新
- ✅ 结构化数据展示

### 常见问题及回答

**Q1: 为什么不生成新PPT，而是更新现有PPT？**
```
A: 因为教师通常已经有自己的课件体系和教学风格，
   完全重新生成会破坏这种连贯性。
   我们的系统只更新案例和数据，保留教师的风格。
```

**Q2: 与其他AI教学系统有什么区别？**
```
A: 主要有3个差异化点：
   1. 课件更新 vs 课件生成
   2. 实验指导 vs 试题生成
   3. 主观题批阅 vs 客观题批阅
   
   这些功能填补了现有系统的空白。
```

**Q3: AI批阅主观题的准确性如何保证？**
```
A: 我们采用"AI辅助+教师审阅"的模式：
   1. AI提供初步评分和评语
   2. 教师可以修改评分和补充评语
   3. 最终成绩由教师确认
   
   这样既提高了效率，又保证了准确性。
```

**Q4: 系统的实用性如何？**
```
A: 非常实用：
   1. 教师每学期都需要更新课件案例
   2. 实践类课程需要详细的实验指导书
   3. 主观题批阅是最耗时的工作
   
   我们的系统可以将这些工作的时间缩短80%以上。
```

## 📞 常见问题

### Q: 数据库初始化失败
**解决方法**：
1. 检查MySQL密码是否正确
2. 确认web_experiment数据库已存在
3. 检查MySQL服务是否启动

### Q: 后端启动失败
**解决方法**：
1. 检查8080端口是否被占用
2. 确认数据库连接配置正确
3. 检查通义千问API Key是否配置

### Q: 前端无法访问
**解决方法**：
1. 确认前端服务已启动
2. 检查5173端口是否被占用
3. 清除浏览器缓存

### Q: AI功能不工作
**解决方法**：
1. 检查通义千问API Key配置
2. 确认网络连接正常
3. 查看后端日志获取详细错误
