# 🔧 修复考核类型显示问题

## 问题原因
1. 浏览器localStorage中缓存了旧的examConfig配置（type: 'choice'）
2. 页面加载时会从localStorage恢复配置，覆盖了默认值

## ✅ 已修复的问题
1. ✅ examConfig的默认type已改为'exam'
2. ✅ goToStep方法已修复，使用examConfig.value.type而不是examConfig.type

## 🔧 清除缓存步骤

### 方法1：浏览器开发者工具清除（推荐）
1. 按F12打开开发者工具
2. 切换到"Application"（应用）或"存储"标签
3. 左侧找到"Local Storage"
4. 点击你的网站域名
5. 找到并删除以下键：
   - `examConfig`
   - `generatedExam`
   - `temp_generated_exam`
6. 刷新页面（Ctrl+F5 或 Cmd+Shift+R）

### 方法2：在浏览器控制台执行
1. 按F12打开开发者工具
2. 切换到"Console"（控制台）标签
3. 输入以下命令并回车：
```javascript
localStorage.removeItem('examConfig')
localStorage.removeItem('generatedExam')
sessionStorage.removeItem('temp_generated_exam')
location.reload()
```

### 方法3：清除所有网站数据
1. 按F12打开开发者工具
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

## ✅ 清除缓存后的效果
- 考核类型默认显示"考试"
- 选择"实验指导书"类型，点击"生成实验指导书"按钮，直接调用AI生成
- 选择"大作业"类型，点击"生成大作业"按钮，直接创建大作业
- 选择"考试"类型，点击"下一步：配置题目"按钮，进入第二步

## 🎯 验证步骤
1. 清除缓存后刷新页面
2. 检查"考核类型"下拉框，应该默认显示"考试"
3. 切换到"实验指导书"，填写信息，点击按钮测试
4. 切换到"大作业"，填写信息，点击按钮测试

## 📝 注意事项
- 如果清除缓存后仍显示"choice"，请尝试硬刷新（Ctrl+Shift+R）
- 确保浏览器没有禁用JavaScript
- 如果问题仍然存在，请检查浏览器控制台是否有错误信息
